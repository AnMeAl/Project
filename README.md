# ИИ-ассистент для подбора съемного жилья

Сервис для автоматической проверки объявлений о аренде квартир. Система получает на вход текстовое описание желаемой квартиры, находит семантически похожие объявления, анализирует рынок и выдаёт структурированный отчёт с рекомендацией.

## Цель проекта

Разработать ИИ-помощника, который помогает пользователям быстро найти подходящую квартиру, оценить адекватность цены и получить объективную рекомендацию на основе рыночных данных.

## Как это работает

### Основные этапы:
1. **Ввод данных** — пользователь описывает желаемую квартиру в свободной форме
2. **Векторизация** — текст запроса преобразуется в эмбеддинг (312 чисел)
3. **Поиск** — FAISS находит 50 семантически похожих квартир в базе
4. **Фильтрация** — из запроса извлекаются числовые параметры (комнаты, цена, этаж, метраж)
5. **Отбор** — выбираются топ-5 квартир с учётом жёстких фильтров
6. **Анализ** — GPT-4o-mini анализирует найденные варианты
7. **Отчёт** — пользователь получает таблицу сравнения и итоговую рекомендацию

## Итоговая реализация

### 1. Сбор данных
- **Источник:** Яндекс.Недвижимость
- **Объём:** 2078 квартир (1–4 комнаты, Москва)
- **Типы данных:**
  - Табличные: площадь, комнаты, этаж, этажность, цена
  - Текстовые: адрес, описание
  - Изображения: до 5 фото на квартиру

### 2. Очистка данных (Pandera)
- Приведение типов (float/int)
- Валидация значений (цены 10k–100M ₽, площадь 10–1000 м²)
- Удаление аномалий (этаж не может быть больше этажности)
- Обработка проблем кодировки (UTF-8)

### 3. Признаки для модели
| Тип | Признаки | Размерность |
|-----|----------|-------------|
| Числовые | Площадь, комнаты, этаж, этажность, Этаж_норм | 5 |
| Текстовые | rubert-tiny2 (адрес + описание) | 312 |
| Визуальные | CLIP (усреднение 5 фото) | 512 |

### 4. Сравнение моделей CatBoost
| Модель | MAE (тыс. ₽) | R² | MAPE (%) |
|--------|--------------|-----|----------|
| Только числовые | 24.5 | 0.571 | 23.4 |
| Числовые + текст | 20.2 | 0.685 | 19.2 |
| Числовые + текст + изображения | 20.5 | 0.672 | 19.6 |

**Вывод:** Наибольший эффект даёт добавление текстовых эмбеддингов (улучшение R² на 20%). Визуальные эмбеддинги не дали значительного прироста из-за малого размера выборки и усреднения 5 фото.

### 5. Архитектура поиска
- **FAISS индекс:** 2078 векторов размерностью 824 (текст + изображения)
- **Гибридный поиск:** семантический (FAISS) + жёсткие фильтры (комнаты, цена, этаж)
- **Извлечение параметров:** регулярные выражения для парсинга чисел из текста

### 6. LLM-генерация отчёта
- **Модель:** GPT-4o-mini (через ProxyAPI)
- **Формат ответа:** таблица сравнения + сводка + итоговая рекомендация
- **Время ответа:** 2–4 секунды

## Запуск проекта

### 1. Клонирование репозитория

```
git clone https://github.com/AnMeAl/Project.git
cd Project
```

### 2. Установка uv (пакетный менеджер)

```
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 3. Установка зависимостей

```
uv sync
```

### 4. Настройка переменных окружения

```
cp .env.example .env
# Отредактируйте .env, добавьте токены
```

### 5. Запуск MinIO (для изображений)

```
docker run -d \
  --name minio \
  -p 9000:9000 \
  -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -v ~/minio-data:/data \
  minio/minio server /data --console-address ":9001"
```

### 6. Загрузка данных через DVC

```
dvc pull
```

### 7. Запуск веб-интерфейса

```
streamlit run app.py
```
